Product Requirements Document (PRD): AI -Driven Fleet Management & Route Optimization 
Module  
1. Product Overview & Vision  
The goal of this module is to upgrade the existing "GREENIE Smart C&D Waste Monitoring System" 
by integrating rigorous Artificial Intelligence (AI) and Machine Learning (ML) models. While the 
current platform successfully provides real -time geospatial trac king, fleet registries, and operational 
dashboards, introducing AI will transition the system from a passive tracking tool into an active, 
predictive, and autonomous logistics engine. The focus will be on dynamic route optimization, load 
matching, and pred ictive maintenance.  
2. Baseline System Analysis (Based on Current UI)  
The AI integration will build upon the existing modules visible in the system:  
• Operations Dashboard:  Currently tracks 124 total units (84 active, 22 idle, 17 in 
maintenance) and pending collections (e.g., 23 awaiting dispatch).  
• Collection Sites:  Manages node capacities (e.g., CS -001 Chennai Central Hub at 50 tons, CS -
002 Coimbatore North Depot at 80 tons).  
• Monitoring / Map:  Provides static live -feed plotting of active, idle, and maintenance sites 
across South India.  
• Vehicles Registry:  Logs specific vehicle data including plate numbers, models (Tata 407, 
Ashok Leyland Dost, BharatBenz 1617R), current mileage, and last service dates.  
• Analytics & Reports:  Tracks revenue (e.g., ₹659K), average collections per site/day (14.2), 
and waste by type.  
3. Proposed AI Features & Functional Requirements  
Feature 1: Dynamic Route Optimization & Dispatch (AI -Routing)  
• Requirement:  The system must autonomously assign the pending collections (currently 
shown as 23 on the dashboard) to the 22 idle units using AI -driven dispatching.  
• AI Model:  Implement Deep Q -Networks (DQN) and Reinforcement Learning (RL) models to 
solve the Capacitated Vehicle Routing Problem (CVRP).  
• Functionality:  
o The algorithm must dynamically respond to real -time traffic conditions, weather 
forecasts, and road closures, recalculating the optimal path continuously.  
o It must perform "eco -routing" to minimize harsh accelerations and gradients, 
thereby reducing fuel burn and carbon emissions.  
• UI Integration:  The Monitoring / Map  module must be updated to display AI -generated 
dynamic route lines (via Mapbox/OpenStreetMap integration) rather than just point -in-time 
vehicle locations.  
Feature 2: AI -Assisted Load & Capacity Matching  
• Requirement:  Maximize the utilization of the fleet based on specific vehicle capacities and 
waste types.  
• AI Model:  Utilize Fractional Knapsack dynamic programming paired with Deep Q -Networks 
to maximize vehicle capacity utilization up to 100%.  
• Functionality:  The AI must match heavy payloads (e.g., concrete and rubble) with heavy 
trucks like the Mahindra Furio or BharatBenz 1617R (from the Vehicles  registry), while 
routing lighter trucks like the Tata 407 for smaller, mixed -debris pickups.  
Feature 3: Predictive Fleet Maintenance  
• Requirement:  Reduce the number of vehicles in maintenance (currently at 17, representing a 
significant portion of the 124 total units) through preemptive servicing.  
• AI Model:  Deploy component -level health models using historical telematics and ML 
algorithms.  
• Functionality:  
o Analyze real -time vibration, temperature, and voltage data alongside the existing 
mileage and "last service" dates in the Vehicles  registry.  
o Flag anomalies and predict component failures before roadside breakdowns occur.  
• UI Integration:  Add an "AI Health Score" column to the Vehicles  table and generate 
automated alerts in the Operations Dashboard  for prescriptive slotting (booking 
maintenance during route gaps to preserve utilization).  
4. Data & Technical Requirements  
• Data Inputs:  Real -time GPS telematics, traffic data APIs (e.g., Google Maps or SUMO for 
traffic simulation), vehicle specifications, and load cell weights from the transfer stations.  
• Backend Architecture:  A scalable Python/Flask backend to run the CVRP algorithms and 
process the RL models without lagging the frontend dashboard.  
• Hardware Integration:  Ensure compatibility with existing IoT sensors on the trucks and the 
transfer station's weighbridges to feed accurate payload data into the routing algorithm.  
5. Success Metrics (KPIs)  
To prove the viability and profitability of this AI integration to investors, the system will target the 
following metrics:  
• Distance & Fuel Reduction:  Achieve a 25% to 30% reduction in total travel distance and fuel 
consumption compared to baseline heuristic routing methods.  
• Fleet Utilization:  Increase total vehicle capacity utilization to near 100%, allowing the same 
volume of waste (e.g., 355 T) to be collected with up to 20% fewer deployed vehicles.  
• Downtime Reduction:  Decrease the percentage of the fleet "In Maintenance" (currently at 
~13%) by at least half through predictive rather than reactive repairs.

UI/UX Design Document: GREENIE Autonomous Fleet Management System  
1. Design Vision & Concept  
The objective is to evolve the existing GREENIE dashboard from a static tracking portal into a 
dynamic, autonomous -ready command center. By fusing the structural layout of your current system 
(Images 3 –7) with the advanced telemetry of autonomous vehicle i nterfaces (like Fleeet.so), the UI 
will allow dispatchers to monitor AI -driven routes, view real -time vehicle camera feeds, and track 
electric/autonomous vehicle battery levels seamlessly.  
2. Visual Language & Design System  
Based on your uploaded UI screenshots, the system will retain its clean, professional aesthetic while 
introducing high -tech elements suited for AI operations.  
• Color Palette:  
o Primary Base:  Deep Navy Blue (#1E293B) for the left sidebar and top navigation to 
maintain brand consistency.  
o Background:  Off-white/Light Gray (#F8FAFC) for the main workspace to ensure high 
contrast for data cards.  
o Accents:  Gold/Yellow (#F59E0B) for active tab states and primary action buttons.  
o Status Indicators (Semantic):  Emerald Green (Active/Charging), Amber (Idle/Low 
Battery), Rose Red (Maintenance/AI Alert).  
o Theme Toggle:  Introduce a Dark Mode  (dark slate backgrounds with neon accents), 
which is an industry standard for autonomous monitoring and reduces eye strain in 
control rooms.  
• Typography & Layout:  
o Clean, sans -serif fonts (like Inter or Roboto) for high legibility of data grids.  
o Modular Card System:  Keep the existing rounded -corner white cards with subtle 
drop shadows (as seen in Image 4 and 6) to house individual widgets.

1. Hardware & IoT Edge Stack (Transfer Station Segregation)  
This layer handles the physical sorting of waste and data transmission from the transfer stations, 
aligning with your hardware documentation.  
• Microcontrollers:  ESP32 -CAM and ESP32 -S3 for edge processing and controlling the 
segregation logic.  
• Vision Sensors:  OV2940 Camera Module for continuous image capture on the conveyor belt.  
• Physical Sensors:  
o IR/Ultrasonic proximity sensors to detect waste placement.  
o Load Cell with an HX711 amplifier to measure the weight of the incoming C&D 
waste.  
• Actuators & Motor Control:  Servo motors to actuate the pneumatic sorting gates, driven by 
TB6612 or A4888 motor drivers.  
• Connectivity:  Native Wi -Fi via the ESP32, with an optional SIM800L GSM Module for remote 
transfer stations lacking stable broadband.  
2. AI & Machine Learning Stack  
Based on Image 2  (which shows a live object detection script running locally) and your project 
documents, the AI stack must support real -time video inference and predictive routing.  
• Computer Vision Frameworks:  Python and OpenCV. The interface in Image 2  is a classic 
OpenCV (cv2.imshow) window running a Python script, identifying objects like "hard plastic," 
"timber," and "aggregate" with bounding boxes and confidence scores.  
• Model Training & Deployment:  
o Roboflow:  For dataset annotation, augmentation, and managing your computer 
vision pipelines.  
o Edge Impulse:  For converting and deploying lightweight Convolutional Neural 
Network (CNN) models directly onto the ESP32 edge devices.  
o Kaggle:  For sourcing initial training datasets for C&D materials.  
• Object Detection Algorithm:  YOLOv8 (or a similar lightweight CNN). It provides the high FPS 
(Frames Per Second) and accuracy required for conveyor belt sorting as seen in your 14.8 FPS 
readout in Image 2.  
• Fleet Routing AI:  Deep Q -Networks (DQN) and Reinforcement Learning (built with PyTorch or 
TensorFlow) running on the backend to dynamically solve the Capacitated Vehicle Routing 
Problem (CVRP) for your trucks.  
3. Frontend Web Dashboard (Fleet & Station Management)  
Based on the visual design language in Images 3 through 7 , the web platform requires a modern, 
component -driven frontend to render data cleanly.  
• Core Framework:  React.js or Next.js. This allows for the modular card -based UI (e.g., the 
"Total Units" and "Revenue Generated" cards) seen across your dashboard.  
• Styling:  Tailwind CSS. The clean typography, precise padding, and color -coded status badges 
(Active/Idle/Maintenance) in the images strongly suggest a utility -first CSS framework.  
• Maps & Geospatial Data:  Image 5  clearly displays the watermark "Leaflet | © 
OpenStreetMap"  in the bottom right corner. Leaflet.js combined with OpenStreetMap tiles is 
perfect for rendering the live site map and plotting truck locations.  
• Data Visualization:  Chart.js or Recharts. Required to build the smooth "Weekly Collection 
Trend" line graphs and the "Fleet Status Distribution" doughnut charts visible in Image 4 , as 
well as the "Waste by Type" bar charts in Image 6 . 
4. Customer & Contractor Mobile Application  
For the application where contractors can book and track C&D waste pickups:  
• Framework:  Flutter or React Native. This allows you to build for both iOS and Android 
simultaneously using a single codebase.  
• Maps API:  Google Maps API or Mapbox SDK for mobile to allow contractors to drop a pin on 
their construction site and track the ETA of your drivers live.  
5. Backend & Cloud Infrastructure  
To bridge the heavy AI calculations, the IoT hardware, and the web dashboards:  
• Backend Server:  Python (Flask or FastAPI) or Node.js (Express). A Python backend is highly 
recommended to seamlessly integrate your existing Python -based AI object detection scripts 
and reinforcement learning routing algorithms.  
• Database (Relational):  PostgreSQL to store structured data like contractor accounts, vehicle 
registries, driver profiles, and billing information.  
• Database (Time -Series):  InfluxDB or MongoDB to handle the high -velocity stream of sensor 
data, load cell weights, and live GPS coordinates from the trucks.  
• Real -Time Communication:  WebSockets (Socket.io). Essential for pushing live updates to the 
dashboard without page refreshes, ensuring the Map pins (Image 5) and the Operations 
Dashboard numbers (Image 4) tick upwards in real -time as waste is processed.
